# 数据对比和批量操作使用指南

**版本：** v1.6.0+  
**最后更新：** 2026-03-20

## ⚖️ 数据对比工具

### 表结构对比

**使用场景：**
- 数据库迁移前验证
- 开发/生产环境对比
- 版本升级对比
- 代码审查辅助

**对比步骤：**
1. 导航到"对比"页面
2. 选择要对比的表
3. 点击"开始对比"
4. 查看对比结果
5. 导出对比报告

**对比内容：**
- 新增列 (绿色标识)
- 删除列 (红色标识)
- 修改列 (黄色标识)
  - 类型变化
  - 约束变化
  - 默认值变化

### 数据对比

**使用场景：**
- 数据迁移验证
- 数据同步检查
- 数据一致性验证
- 备份恢复验证

**对比配置：**
- 键列选择 (必填)
- 对比列选择 (可选)
- 忽略列设置 (可选)

**对比结果：**
- 新增记录 (INSERT)
- 删除记录 (DELETE)
- 修改记录 (UPDATE)

## 🔧 批量操作

### 批量执行 SQL

**使用步骤：**
1. 打开查询页面
2. 找到"批量执行"组件
3. 输入 SQL 语句 (用分号分隔)
4. 点击"解析 SQL"
5. 确认操作列表
6. 点击"执行"

**支持的 SQL 类型：**
- SELECT (查询)
- INSERT (插入)
- UPDATE (更新)
- DELETE (删除)
- 任意有效 SQL

**执行选项：**
- 遇到错误停止 (可选)
- 事务执行 (待实现)
- 分块执行 (待实现)

### 批量生成 SQL

**批量 INSERT：**
```sql
-- 生成语句
INSERT INTO users (id, name, email) VALUES
(1, 'Alice', 'alice@example.com'),
(2, 'Bob', 'bob@example.com'),
(3, 'Charlie', 'charlie@example.com');
```

**批量 UPDATE：**
```sql
-- 生成语句
UPDATE users SET status = 'active' WHERE id = 1;
UPDATE users SET status = 'active' WHERE id = 2;
UPDATE users SET status = 'active' WHERE id = 3;
```

**批量 DELETE：**
```sql
-- 生成语句
DELETE FROM users WHERE id IN (1, 2, 3, 4, 5);
```

## 💡 使用技巧

### 表结构对比技巧

**对比多个表：**
1. 逐个表进行对比
2. 导出所有报告
3. 汇总分析

**生成迁移脚本：**
1. 对比表结构
2. 导出 SQL 脚本
3. 在目标数据库执行

### 批量操作技巧

**准备 SQL 列表：**
```sql
-- 从 Excel 生成
SELECT CONCAT('UPDATE users SET status = ''active'' WHERE id = ', id, ';')
FROM users WHERE status = 'pending';

-- 复制生成的 SQL 到批量执行框
```

**分批次执行：**
1. 每批 100-500 条 SQL
2. 执行完一批再执行下一批
3. 避免单次执行过多

**错误处理：**
1. 启用"遇到错误停止"
2. 查看错误信息
3. 修复后继续执行

## ⚠️ 注意事项

### 性能建议

**表结构对比：**
- 建议 < 100 个列
- 复杂表先导出再对比

**数据对比：**
- 建议 < 10,000 行
- 使用索引列作为键列
- 避免对比大文本列

**批量操作：**
- 每批 < 1,000 条 SQL
- 大数据量分批执行
- 避免长时间锁表

### 安全建议

**数据对比：**
- 只读权限即可
- 避免在生产环境对比大表
- 注意数据隐私

**批量操作：**
- 先在测试环境验证
- 备份重要数据
- 使用事务 (待支持)
- 谨慎执行 DELETE/TRUNCATE

## 🐛 常见问题

### Q: 对比结果显示不一致但实际相同？

**A:** 检查：
1. 数据类型是否完全一致
2. 默认值格式是否相同
3. 约束名称是否一致

### Q: 批量执行部分失败？

**A:** 处理：
1. 查看失败原因
2. 导出失败记录
3. 修复后重新执行

### Q: 导出报告乱码？

**A:** 确保：
1. 使用 UTF-8 编码
2. 浏览器支持中文
3. 文件用正确编码打开
