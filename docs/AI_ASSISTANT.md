## 📖 SQL 解释模式

### 使用场景

- 理解复杂 SQL
- 学习 SQL 语法
- 代码审查

### 使用示例

**输入：**
```sql
SELECT u.name, COUNT(o.id)
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.status = 'active'
GROUP BY u.id
ORDER BY COUNT(o.id) DESC
LIMIT 10
```

**输出：**
- **总结：** 查询活跃用户的订单数量
- **分解：**
  - SELECT: 选择用户名称和订单数
  - FROM: 从 users 表
  - LEFT JOIN: 连接 orders 表
  - WHERE: 过滤活跃用户
  - GROUP BY: 按用户分组
  - ORDER BY: 按订单数排序
  - LIMIT: 限制 10 条

## ⚡ SQL 优化模式

### 使用场景

- 性能优化
- 代码审查
- 最佳实践

### 使用示例

**输入：**
```sql
SELECT * FROM users WHERE name LIKE '%test%'
```

**输出：**
- **优化后：** `SELECT column1, column2 FROM users WHERE ... LIMIT 1000`
- **建议：**
  - 避免 SELECT *，明确指定需要的列
  - 避免前缀通配符 LIKE，考虑使用全文索引
  - 添加 LIMIT 限制返回行数
- **性能提升：** 预计 60%
