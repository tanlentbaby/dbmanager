## 📦 新增功能

### 1. 🔍 SQL 错误诊断 - `/diagnose`

**功能描述：**
自动诊断 SQL 执行错误，提供详细的解决方案和置信度评分。

**使用示例：**
```bash
# 诊断错误消息
/diagnose "table 'users' doesn't exist"

# 诊断错误代码
/diagnose 1064
/diagnose 1146

# 自动诊断（SQL 执行出错时自动显示）
sql> SELEC * FROM users;
# 自动显示诊断报告
```

**支持的数据库：**
- MySQL: 1045/1049/1054/1062/1064/1146/1205/2003/2006
- PostgreSQL: 28P01/3D000/42703/42P01/23505/23503/57014
- SQLite: SQLITE_ERROR/SQLITE_INTERNAL/SQLITE_PERM/SQLITE_BUSY/SQLITE_LOCKED/SQLITE_FULL

**输出示例：**

### 2. ⚡ 查询优化建议 - `/optimize`

**功能描述：**
分析 SQL 查询结构，检测潜在性能问题并提供优化建议。

**使用示例：**
```bash
/optimize SELECT * FROM users
/opt SELECT * FROM orders WHERE user_id IN (SELECT id FROM users)
```

**检测项（10+ 种）：**

| 严重程度 | 检测项 | 说明 |
|---------|--------|------|
| 🔴 Critical | UPDATE/DELETE 无 WHERE | 会更新/删除所有数据 |
| 🔴 Critical | JOIN 缺少 ON 条件 | 可能产生笛卡尔积 |
| 🟡 Warning | SELECT * 使用 | 获取不必要的列 |
| 🟡 Warning | 缺少 WHERE/LIMIT | 可能返回大量数据 |
| 🟡 Warning | LIKE 前缀通配符 | 导致索引失效 |
| 🟡 Warning | 函数包裹列 | 导致索引失效 |
| 🟢 Info | 子查询优化 | 可改用 JOIN/EXISTS |
| 🟢 Info | OR 条件优化 | 可改用 IN/UNION |
| 🟢 Info | NOT IN 优化 | 性能较差 |
| 🟢 Info | ORDER BY 无 LIMIT | 对所有结果排序 |

**输出示例：**
```
╔══════════════════════════════════════════════════════════╗
║  ⚡ SQL 查询优化建议报告                                  ║
╚══════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────
│ 发现 2 个优化点
├─────────────────────────────────────────────────────────
│
│ 1. 🟡 [选择] 使用 SELECT * 会获取所有列，可能影响性能
│    建议：明确指定需要的列名，如：SELECT id, name, email FROM ...
│    置信度：90%
│
│ 2. 🟡 [条件] 查询没有 WHERE 条件，可能返回大量数据
│    建议：添加 WHERE 条件限制结果集，或使用 LIMIT 限制行数
│    置信度：80%
│
└─────────────────────────────────────────────────────────
```

### 3. 🗣️ 自然语言生成 SQL - `/nl2sql`

**功能描述：**
将中文自然语言转换为 SQL 语句，降低 SQL 使用门槛。

**使用示例：**
```bash
# 简单查询
/nl2sql 查询所有用户

# 带条件
/nl 查找年龄大于 25 的用户

# 统计
/nl 统计订单数量

# 排序
/nl 显示前 10 个订单，按创建时间降序排序

# DELETE
/nl 删除 id 等于 1 的用户
```

**支持的操作：**
- 查询：查询、查找、搜索、显示、列出、查看
- 统计：统计、计数、多少个
- 条件：等于、大于、小于、包含
- 排序：升序、降序、按...排序
- 限制：前 N 条、限制 N 条

**输出示例：**
```
╔══════════════════════════════════════════════════════════╗
║  🗣️ 自然语言生成 SQL                                     ║
╚══════════════════════════════════════════════════════════╝

📝 输入：查找年龄大于 25 的用户，按创建时间降序排序，显示前 10 条

┌─────────────────────────────────────────────────────────
│ 生成的 SQL
├─────────────────────────────────────────────────────────
│ SELECT * FROM users WHERE 年龄 > 25 ORDER BY created_at DESC LIMIT 10
└─────────────────────────────────────────────────────────

📊 置信度：85%
💡 说明：查询 users 表，条件：1 个，排序：created_at，限制 10 条
```

### 5. 🔖 书签管理系统

**功能描述：**
保存常用查询，支持标签分类、搜索和快速调用。

**使用示例：**
```bash
# 添加书签
/bookmark add "用户列表" "SELECT * FROM users" mysql users

# 列出书签
/bookmark list
/bookmark list mysql          # 按标签过滤

# 搜索书签
/bookmark search user

# 运行书签
/bookmark run "用户列表"
/bm run "订单统计"            # 快捷命令

# 查看标签
/bookmark tags

# 导出/导入
/bookmark export
/bookmark import bookmarks.json

# 统计信息
/bookmark stats
```

**功能特性：**
- 🔖 添加/删除/更新书签
- 🏷️ 按标签分类管理
- 🔍 搜索书签（名称/SQL/描述/标签）
- ▶️ 快速运行书签查询
- 📤 导出/导入（JSON 格式）
- 📊 使用统计（次数/最后使用时间）
- 📦 内置书签（常用查询模板）

## 🐛 已知问题

1. **NL2SQL 列名提取**
   - 当前版本：WHERE 子句可能包含多余词汇
   - 示例：`WHERE 查找年龄 > 25` 应为 `WHERE 年龄 > 25`
   - 计划：v0.5.1 修复

2. **NL2SQL 复杂查询**
   - 当前版本：规则引擎，支持简单到中等复杂度
   - 计划：v0.6.0 集成 LLM，支持更复杂查询

3. **Web UI 集成**
   - 智能功能暂未集成到 Web UI
   - 计划：v0.5.1 集成
