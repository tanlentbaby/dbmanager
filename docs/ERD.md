## 🗺️ ER 图功能

### 什么是 ER 图

ER 图 (Entity-Relationship Diagram) 是数据库设计的可视化工具，用于展示：
- 表结构 (实体)
- 列信息 (属性)
- 表关系 (关系)

### 访问 ER 图

1. 连接数据库
2. 点击导航栏"ER 图"
3. 自动加载表结构
4. 查看可视化 ER 图

---

## 📊 ER 图元素

### 表节点

| 元素 | 说明 | 标识 |
|------|------|------|
| 表头 | 表名称 + Schema | 蓝色渐变背景 |
| 主键列 | 主键字段 | 🔑 金色钥匙 |
| 外键列 | 外键字段 | 🔗 蓝色链接 |
| 列类型 | 数据类型 | 灰色小字 |
| 非空列 | 必填字段 | * 红色星号 |
| 注释 | 列注释 | 💬 气泡图标 |

### 关系连线

| 元素 | 说明 | 样式 |
|------|------|------|
| 外键关系 | 表之间的引用 | 灰色实线 |
| 箭头 | 引用方向 | 箭头指向被引用表 |
| 标签 | 列名映射 | 列名 → 引用列名 |

## 🎨 ER 图操作

### 基本操作

**缩放：**
- 鼠标滚轮 - 放大/缩小
- Ctrl + 滚轮 - 快速缩放

**平移：**
- 鼠标拖拽空白处 - 移动画布
- 触摸板双指滑动 - 移动画布

**选择：**
- 点击表节点 - 选中表
- Shift + 点击 - 多选

### 导航

**小地图：**
- 右下角小地图显示全局
- 点击小地图快速定位
- 蓝色框表示当前视图

**控制面板：**
- 左下角控制面板
- 放大/缩小按钮
- 适应屏幕按钮
- 重置按钮

## 📤 导出 ER 图

### 导出格式

| 格式 | 说明 | 使用场景 |
|------|------|----------|
| PNG | 位图图片 | 文档/演示/网页 |
| JPEG | 位图图片 | 文档/演示 |
| PDF | PDF 文档 | 正式文档/打印 |
| SQL | DDL 脚本 | 数据库迁移/备份 |

### 导出步骤

1. 打开 ER 图页面
2. 调整到合适的视图
3. 点击右上角导出按钮
4. 选择导出格式
5. 自动下载文件

### 导出示例

**导出为 PNG：**
```
点击 🖼️ PNG 按钮
下载 erd.png
```

**导出为 PDF：**
```
点击 📄 PDF 按钮
下载 erd.pdf
```

**导出为 SQL：**
```
点击 💾 SQL 按钮
下载 schema.sql
```

## 🔧 高级功能

### 从 DDL 生成 ER 图

**步骤：**
1. 准备 SQL DDL 文件
2. 在查询页面执行 DDL
3. 切换到 ER 图页面
4. 自动显示新生成的表

**支持的 DDL：**
```sql
CREATE TABLE users (
  id INT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE
);

CREATE TABLE posts (
  id INT PRIMARY KEY,
  user_id INT,
  title VARCHAR(255),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 导出 SQL DDL

**导出的 SQL 包含：**
- CREATE TABLE 语句
- 列定义 (名称/类型/约束)
- 主键约束
- 外键约束
- 数据库特定语法 (ENGINE/CHARSET)

**示例输出：**
```sql
CREATE TABLE users (
  id INT NOT NULL,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE,
  PRIMARY KEY (id)
);

CREATE TABLE posts (
  id INT NOT NULL,
  user_id INT NOT NULL,
  title VARCHAR(255),
  PRIMARY KEY (id),
  CONSTRAINT fk_posts_user 
    FOREIGN KEY (user_id) REFERENCES users(id)
);

ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 💡 使用技巧

### 性能优化

**大型数据库：**
- 建议 < 100 个表
- 使用搜索功能定位表
- 导出时选择合适格式

**视图优化：**
- 缩放至合适比例
- 使用小地图导航
- 分组查看相关表

### 最佳实践

**表命名：**
- 使用有意义的表名
- 遵循命名规范
- 添加表注释

**关系设计：**
- 明确外键关系
- 使用约束名称
- 避免循环引用

**文档化：**
- 添加列注释
- 导出为文档
- 定期更新 ER 图

---

## ⚠️ 注意事项

### 性能限制

| 场景 | 建议 | 原因 |
|------|------|------|
| 表数量 | < 100 个 | 渲染性能 |
| 导出图片 | < 50MB | 文件大小 |
| 复杂关系 | 简化设计 | 可读性 |

### 已知限制

1. **自引用关系** - 显示可能不直观
2. **复合主键** - 显示待优化
3. **多对多关系** - 需要中间表
4. **超大 ER 图** - 建议分批查看

## 🐛 常见问题

### Q: ER 图加载慢？

**A:** 检查：
1. 表数量是否过多
2. 数据库连接是否正常
3. 尝试刷新页面

### Q: 关系线显示混乱？

**A:** 建议：
1. 拖动表节点调整位置
2. 使用缩放功能
3. 导出后查看

### Q: 导出图片模糊？

**A:** 尝试：
1. 导出为 PNG 格式 (质量最好)
2. 放大后再导出
3. 使用 PDF 格式

### Q: SQL 导出后无法执行？

**A:** 检查：
1. 数据库类型是否匹配
2. 表顺序是否正确 (先创建被引用表)
3. 是否有循环依赖
