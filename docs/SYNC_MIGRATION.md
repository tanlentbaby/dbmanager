# 数据结构同步和迁移使用指南

**版本：** v1.7.0+  
**最后更新：** 2026-03-20

---

## 🔄 结构同步

### 使用场景

- **开发→生产同步** - 将开发环境的结构变更同步到生产
- **数据库重构** - 修改表结构后同步
- **版本升级** - 应用升级时的结构变更
- **环境一致性** - 保持多环境结构一致

### 同步步骤

**1. 生成同步计划**
1. 导航到"同步"页面
2. 点击"生成同步计划"
3. 系统自动对比源和目标数据库

**2. 预览同步计划**
- 查看需要同步的表
- 查看变更详情 (新增/修改/删除列)
- 查看生成的 SQL 语句

**3. 执行同步**
1. 确认同步计划
2. 点击"执行同步"
3. 等待同步完成

**4. 导出 SQL (可选)**
- 导出 SQL 脚本
- 手动在目标数据库执行
- 用于版本控制和审计

### 同步选项

| 选项 | 说明 | 默认 |
|------|------|------|
| 删除列 | 删除目标表多余的列 | 关闭 |
| 外键同步 | 同步外键约束 | 开启 |
| 索引同步 | 同步索引 | 开启 |

## 📦 数据迁移

### 使用场景

- **数据库迁移** - 迁移到新数据库
- **数据备份** - 备份重要数据
- **数据合并** - 合并多个数据库
- **数据归档** - 归档历史数据

### 迁移步骤

**1. 配置选项**
1. 导航到"迁移"页面
2. 配置迁移选项
   - 选择/排除表
   - 批量大小
   - 清空目标表
   - 禁用约束

**2. 生成迁移计划**
1. 点击"生成迁移计划"
2. 查看表数量和总行数
3. 评估迁移时间

**3. 执行迁移**
1. 确认迁移计划
2. 点击"开始迁移"
3. 实时查看进度

**4. 导出报告**
- 导出迁移报告
- 记录迁移结果
- 用于审计和验证

### 迁移选项

| 选项 | 说明 | 推荐 |
|------|------|------|
| 清空目标表 | 迁移前 TRUNCATE | 按需 |
| 禁用约束 | 临时禁用外键 | 大数据量开启 |
| 批量大小 | 每批行数 | 100-500 |

## 💡 使用技巧

### 结构同步技巧

**小步快跑：**
1. 先同步少量表
2. 验证无误后再全量同步
3. 避免一次性同步过多表

**生成 SQL 审查：**
1. 导出 SQL 脚本
2. DBA 审查
3. 在测试环境执行
4. 生产环境执行

**回滚准备：**
1. 同步前备份目标库
2. 记录同步前结构
3. 准备回滚 SQL

### 数据迁移技巧

**分批迁移：**
1. 大表分批迁移
2. 每批 10-50 万行
3. 避免长时间锁表

**性能优化：**
1. 开启"禁用约束"
2. 增大批量大小
3. 迁移后重建索引

**数据验证：**
1. 迁移后对比行数
2. 抽样对比数据
3. 验证外键关系

## ⚠️ 注意事项

### 性能建议

**结构同步：**
- 建议 < 50 个表
- 大表先同步结构
- 避免业务高峰期

**数据迁移：**
- 建议 < 100 万行/批
- 大表分批次迁移
- 监控数据库负载

### 安全建议

**结构同步：**
- 生产环境先测试
- 备份目标库
- 准备回滚方案

**数据迁移：**
- 只读权限迁移
- 避免生产写入
- 验证数据完整性

## 🐛 常见问题

### Q: 同步失败？

**A:** 检查：
1. 数据库连接是否正常
2. 权限是否足够
3. 是否有锁表
4. 查看错误日志

### Q: 迁移速度慢？

**A:** 优化：
1. 增大批量大小
2. 禁用外键约束
3. 关闭日志
4. 升级网络

### Q: 数据不一致？

**A:** 处理：
1. 重新迁移
2. 对比工具验证
3. 手动修复
