### Phase 1: 自动索引建议 ⭐
**优先级：** 🔴 高  
**预计时间：** 3-5 天

| 功能点 | 描述 | 状态 |
|--------|------|------|
| 查询分析引擎 | 解析 SQL 提取 WHERE/JOIN/ORDER BY 列 | ⬜ |
| 索引建议算法 | 基于查询模式推荐索引类型 | ⬜ |
| 现有索引检测 | 查询数据库现有索引避免重复 | ⬜ |
| 创建语句生成 | 生成 CREATE INDEX 语句 | ⬜ |
| 影响评估 | 预估索引对查询性能的提升 | ⬜ |

**命令：**
```bash
/suggest-index SELECT * FROM users WHERE email = ?
# 输出：建议创建索引 idx_users_email (email)
# 预估性能提升：85% (全表扫描 → 索引扫描)
```

## 💡 技术 notes

### 索引建议算法
- 分析 WHERE 子句中的列
- 分析 JOIN 条件中的列
- 分析 ORDER BY 子句中的列
- 考虑复合索引的可能性
- 避免冗余索引检测

### SQL 修复策略
- 拼写纠错（Levenshtein 距离）
- 表名/列名模糊匹配
- 语法树解析和修正
- 提供多个候选让用户选择

### 云端同步方案
- 方案 A: 自建后端（Node.js + SQLite）
- 方案 B: 使用现有服务（GitHub Gist/Notion API）
- 方案 C: 本地文件 + 用户自选同步工具（iCloud/Dropbox）

**推荐：** 先实现方案 C（最简单），后续可扩展方案 A/B
